核心概念

State

单一状态树，用来保存全部状态

Getter

状态的计算属性

Mutation

更改state的方法，类似事件，可提交对象，需要遵循Vue的响应规则(初始化所需属性/对象添加新属性要使用Vue.set()方法)，必须是同步函数

Action

可执行异步操作、分发(调用)多重mutation、并且action可以相互组合使用(Promise保证组合action的调用顺序)

Module

单一状态树下所有状态集中在一个store对象比较复杂臃肿，因此可以将store分割成module，每个module拥有自己的state/mutation/action/getter/module

  1. 划分模块并引入

  2. 子组件调用模块：模块内部的state默认是局部的属于模块本身，但是模块内部的getters/mutations/actions默认是注册在全局命名空间的，因此子组件可以直接对名称相同的mutation/action进行同步调用，但是子组件调用state需要指定模块名称

  3. 模块调用模块：由于模块内部state是局部的，getters(state,getters,rootState)/actions(state,commit,rootState)可以通过显示指定rootState参数访问根节点的state

  4. 命名空间：模块可以通过添加namespaced使其getter/action/mutation自动根据模块注册的路径调整命名

  5. 命名空间的模块的action的全局注册与分发
